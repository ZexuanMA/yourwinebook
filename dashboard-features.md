### 登入 / 登出

- **URL**：`/login`
- 左側面板：深酒紅品牌區 + 裝飾 emoji
- 右側面板：Email + 密碼表單（密碼可顯示/隱藏）
- 頁面底部列出所有 Demo 帳號，點擊自動填入
- 管理員帳號單獨顯示，有「管理員」標籤
- 登入成功跳轉 `/dashboard`；已登入訪問 `/login` 自動跳轉後台

### Session 機制

- 登入成功後在瀏覽器寫入 `wb_session` Cookie（HttpOnly，有效期 7 天）
- Middleware 讀取 Cookie 進行路由保護：
  - 未登入訪問 `/dashboard/*` → 跳轉 `/login`
  - 非管理員訪問 `/dashboard/admin/*` → 跳轉 `/dashboard`
- 登出清除 Cookie，跳轉 `/login`

## 三、各頁面功能

### 3.1 後台首頁 `/dashboard`

**統計卡（3 個）**
- 上架酒款數
- 最低價酒款數（該酒商在比價中奪冠的數量）
- 平台評分

**平均定價 Banner**
- 顯示該酒商比價酒款的平均售價

**比價酒款明細表格**
- 列：酒款名稱 + 年份、類型（彩色標籤）、你的價格、市場最低價、狀態（最低價 🏆 / 已上架）

---

### 3.2 酒款管理 `/dashboard/wines`

- 頁面頂部顯示：上架款數、最低價款數
- **搜索框**：可按酒名或類型實時過濾
- **表格列**：酒款（帶 emoji 圖標）、類型（彩色 Pill）、產區、你的價格、市場最低價（及超出金額）、比價名次（第 N / 共 M 家）、「查看」按鈕
- 「查看」跳轉前台對應酒款詳情頁（新標籤打開）
- 空狀態：顯示引導提示和「新增酒款」按鈕

---

### 3.3 新增酒款 `/dashboard/wines/new`

分四個卡片區塊的表單：

| 區塊 | 字段 |
|------|------|
| 酒款類型 | 5 個可點選卡片：紅酒 🍷、白酒 🍾、氣泡酒 🥂、玫瑰酒 🌸、甜酒 🍯 |
| 基本資料 | 酒款名稱（必填）、年份（必填）、葡萄品種 |
| 產區資料 | 產區（必填），格式建議「國家 · 地區 · 子產區」 |
| 定價資料 | 售價 HKD（必填）、購買連結 URL |

提交後顯示成功畫面，可選擇「再新增一款」或「返回列表」。

---

### 3.4 流量分析 `/dashboard/analytics`

**6 個統計卡**（均附帶與上週期對比的漲跌百分比）
- 頁面瀏覽量
- 獨立訪客
- 跳出率（降低為正向，顏色反轉）
- 平均訪問時長
- 比價點擊數（用戶點擊「前往購買」的次數）
- 總會話數

**30 天趨勢折線圖**
- 頁面瀏覽量 vs 獨立訪客，雙線對比

**熱門頁面表格**
- 按瀏覽量排序，顯示平均時長和跳出率（跳出率低顯示綠色）

**流量來源進度條**
- 直接訪問、Google 搜索、社交媒體、推薦連結、其他

**裝置類型進度條**
- 手機 📱、桌面 💻、平板 📟

**熱門酒款**
- 按瀏覽量排序，附帶「點擊購買轉化率」

**本週每日柱狀圖**
- 最近 7 天，頁面瀏覽 vs 獨立訪客雙柱對比

**實時訪問記錄**
- 地區（含國旗）、裝置圖標、入口頁面、訪問時長、瀏覽頁數、時間

---

### 3.5 帳號設置 `/dashboard/account`（酒商 & 管理員通用）

**帳號概覽卡**
- 名稱首字母頭像、顯示名稱、Email、入駐日期、帳號狀態
- 管理員顯示「管理員」盾牌徽章

**基本資料表單**
- 顯示名稱（可改）、Email（唯讀，不可修改）
- 聯繫電話、網站連結
- 商家簡介（僅酒商帳號顯示）

**修改密碼**
- 當前密碼、新密碼、確認新密碼（三個輸入框，各有顯示/隱藏按鈕）
- 前端驗證：當前密碼是否正確、新密碼長度不少於 6 位、兩次輸入是否一致
- 成功後顯示「密碼已更新」提示，自動 3 秒後消失

**危險操作（僅酒商帳號顯示）**
- 「暫停帳號」按鈕：暫停後酒款從比價列表隱藏

---

### 3.6 酒商帳號管理 `/dashboard/admin/accounts`（管理員專用）

**4 個篩選統計卡**
- 全部帳號、正常運營、已停用、待審核
- 點擊可快速過濾列表

**搜索框**
- 按酒商名稱或 Email 過濾

**帳號表格**
- 列：酒商（帶首字母頭像 + 網站連結）、Email、上架酒款數、評分、狀態標籤、入駐日期、操作按鈕
- **可展開行**：點擊「詳情」展開，顯示聯繫資料、平台數據、商家簡介
- **操作按鈕**：
  - 正常帳號：顯示「停用」按鈕
  - 已停用帳號：顯示「啟用」按鈕
  - 待審核帳號：顯示「審核通過」按鈕
- 狀態標籤：正常（綠）、已停用（紅）、待審核（橙）

---

### 3.7 入駐申請管理 `/dashboard/admin/applications`（管理員專用）

**狀態標籤篩選**
- 全部、待處理、已聯繫、已批准、已拒絕（帶各狀態數量）

**搜索框**
- 按公司名、聯繫人或 Email 過濾

**申請卡片（每條申請一張卡）**
- 公司名首字母頭像
- 公司名、聯繫人、酒款數量
- 狀態標籤、提交時間
- 聯繫資料 Pill（Email 可直接點擊發郵件、電話、網站）
- 可展開查看「申請說明」
- **底部操作欄**，根據當前狀態顯示不同按鈕：
  - 待處理：「標記已聯繫」、「拒絕申請」、「批准入駐」
  - 已聯繫：「拒絕申請」、「批准入駐」
  - 已批准 / 已拒絕：「重置為待處理」

## 四、技術說明

### 路由結構

```
/login                          登入頁
/dashboard                      後台首頁（酒商視圖）
/dashboard/wines                酒款管理列表
/dashboard/wines/new            新增酒款表單
/dashboard/analytics            流量分析
/dashboard/account              帳號設置
/dashboard/admin/accounts       酒商帳號管理（管理員專用）
/dashboard/admin/applications   入駐申請審核（管理員專用）
```

### API

```
POST /api/auth/login    登入，成功寫入 wb_session Cookie
POST /api/auth/logout   登出，清除 Cookie
GET  /api/auth/me       返回當前登入帳號資料（含 role 字段）
```

### 重要注意事項

`/dashboard` 和 `/login` 在 `[locale]` 路由之外，必須在各自目錄建立獨立的 `layout.tsx`，包含 `<html>`、`<body>`、字體變量（DM Sans + Noto Sans TC）和 `globals.css` 導入。否則頁面將沒有任何樣式。
