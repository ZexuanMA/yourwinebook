# Your Wine Book — 項目規劃與執行路線圖

---

## 1. 項目理解

### 這是什麼平台
- 一個面向香港本地消費者的**葡萄酒決策輔助平台**
- 核心身份：不是酒商，不是電商，而是**買酒前的第一站**
- 產品定位：「生活方式媒體 + 消費決策工具」的混合體
- 同時也是葡萄酒文化的**入口橋樑**——不止比價，還傳遞風土人情

### 核心商業邏輯
- **前端**：以內容、比價、推薦吸引消費者注意力與信任
- **中端**：通過輕量交易入口（MVP 階段跳轉至酒商）完成轉化
- **後端**：匯聚本地酒商，提供低成本獲客渠道與展示空間，未來延伸至平台內交易、履約與數據服務

### 解決的三個核心問題

| 問題 | 用戶語言 | 平台價值 |
|---|---|---|
| 買貴了 | 「同一支酒，銅鑼灣賣 $380，網上賣 $220，到底哪個靠譜？」 | 價格透明化、比價聚合 |
| 買錯了 | 「朋友生日想送支酒，完全唔識揀」 | 場景化推薦、簡明解說、文化入門 |
| 買得煩 | 「格價要開五個 app，仲要擔心係咪假嘢」 | 一站式信息整合、信任背書 |

### 平台的成功標準
- 消費者：**降低決策成本**，感受到「這裡幫我選，比我自己選更好」
- 酒商：**降低獲客成本**，有一個展示自己品牌和庫存的空間
- 平台：**成為香港買酒決策的默認入口**

## 2. 關鍵策略問題（已確認）

### A. 品牌人格 ✅
**「可靠的編輯」為主體 +「朋友感」為輔**
- 官方調性：克制、專業、有品質感，不跳脫
- 社區空間：開放、有人情味，讓用戶自由分享交流
- 平台也是葡萄酒文化的橋樑——背後有風土人情，不止口感

### B. 用戶優先級 ✅
**雙線並行**
- 入門小白和輕度愛好者同時服務
- 首頁用不同區塊分別觸達兩類用戶
- 酒商作為供給側同步建設

### C. 信任機制 ✅
**價格透明 + 用戶口碑**
- 核心信任來自數據——同一支酒在不同酒商的真實價格對比，不藏價
- 輔助信任來自社區——真實用戶評價和購買反饋
- 平台的中立性靠「透明」體現，而非「背書」

### D. 交易模式 ✅
**先跳轉，後自建**
- MVP 階段：用戶在平台發現和決策，點擊後跳轉到酒商完成交易
- 預留平台內下單功能，後續根據酒商接受度和牌照情況演進
- 需要考慮的限制：酒類銷售牌照、酒商合作意願

### E. 內容策略 ✅
**工具與內容並重（50/50）**
- 每支酒都有故事，每個推薦都帶文化背景
- 工具側：比價、搜索、篩選、推薦
- 內容側：產區故事、品酒指南、風土專題、場景推薦

### F. AI 角色 ✅
**受控的顯性助手**
- AI 輸出嚴格基於平台自有內容和供應商數據
- 給出用戶問題的 Top 3 選項 + 推薦理由
- 不自由發揮，不超出站內數據範圍
- 可接入高端模型 API，但必須以平台內容為輸出邊界

### G. 酒商角色 ✅
**雙向透明展示**
- 「以酒找商」：某款酒 → 顯示所有供應商的價格對比
- 「以商找酒」：某家酒商 → 顯示其門店下所有酒款
- 酒商有獨立的店鋪頁面，用戶清楚知道最終跟誰買

### H. 品牌命名 ✅
**Your Wine Book**（暫定）
- 「Book」的隱喻與「可靠的編輯」人格吻合
- 暗示文化內容屬性——不只是工具，還是一本可以翻閱的酒書
- 中英文皆可使用，國際感強

### I. 視覺方向 ✅
**國際極簡**
- 乾淨、留白多、無襯線字體為主
- 參考 Aesop / Seedlip 的視覺語言（僅作視覺靈感，非品牌模仿）
- 需要在極簡中保留溫度，避免過冷

### J. 語言策略 ✅
**中英雙語**
- 繁體中文 + 英文雙語切換
- 覆蓋香港本地用戶和外籍居民
- 酒名、術語保留英文原文

### K. 社區規劃 ✅
**MVP 保留基本功能，重心在內容和工具**
- MVP 階段：基本的評價/打分功能
- 後期優化：完整社區互動、用戶分享、討論區
- 不在 MVP 階段投入過多社區開發資源

## 3. 工作分解結構（WBS）

### WS-1：品牌策略定位

| 項目 | 內容 |
|---|---|
| **目標** | 確定品牌的核心定位語言、人格、調性刻度 |
| **關鍵任務** | ① 競品調性分析（Wine Searcher / Vivino / 本地酒商 app）② 用戶價值主張提煉 ③ 品牌人格定義（可靠編輯 + 朋友感）④ 定位語句撰寫 ⑤ 調性光譜確認（premium ↔ practical 的精確刻度）⑥ 品牌禁忌清單 |
| **產出** | 品牌定位文件（Brand Positioning Brief） |

### WS-2：用戶旅程與信息架構

| 項目 | 內容 |
|---|---|
| **目標** | 定義雙線用戶的核心路徑，確定首頁信息層級 |
| **關鍵任務** | ① 入門用戶 + 輕度愛好者的 Jobs-to-be-Done 梳理 ② 核心使用場景定義（送禮、聚餐、自飲、囤貨、探索）③ 用戶旅程地圖（entry → discovery → decision → handoff to merchant）④ 首頁信息架構草案 ⑤ 「以酒找商」與「以商找酒」雙路徑設計 ⑥ 導航與分區邏輯 |
| **產出** | 用戶旅程圖 + 首頁線框邏輯（wireframe-level 信息結構） |

### WS-3：品牌視覺系統

| 項目 | 內容 |
|---|---|
| **目標** | 建立國際極簡的視覺語言系統，在 premium 感中保留溫度 |
| **關鍵任務** | ① Moodboard 製作（2-3 個方向）② 色彩系統定義 ③ 字體系統（中英文雙語適配）④ 攝影/插畫風格指引 ⑤ UI 元素風格（卡片、按鈕、標籤、比價表格）|
| **產出** | 品牌視覺指南（Visual Identity Guidelines） |

### WS-4：首頁結構與故事線設計

| 項目 | 內容 |
|---|---|
| **目標** | 設計首頁的敘事節奏，30 秒內讓用戶理解「這是什麼 + 對我有什麼用」 |
| **關鍵任務** | ① Hero section 概念（第一屏核心訊息）② Section-by-section 故事線（工具區 + 內容區交替）③ 每個區塊的功能目標與情緒目標 ④ CTA 策略（主 CTA / 次 CTA / 軟轉化）⑤ 信任元素嵌入（價格透明 + 用戶口碑）⑥ 雙線用戶的分流設計 |
| **產出** | 首頁結構文件（Section Map with intent annotations） |

### WS-5：文案撰寫

| 項目 | 內容 |
|---|---|
| **目標** | 撰寫首頁全部中英文案，確保調性統一、信息清晰、轉化有力 |
| **關鍵任務** | ① 品牌標語（tagline / slogan，中英雙語）② Hero 文案 ③ 各區塊標題與正文 ④ CTA 文案 ⑤ 微文案（tooltip、空狀態、引導語）⑥ SEO 標題與描述 |
| **產出** | 完整首頁文案文件（中英雙語）+ 備選方案 |

### WS-6：核心功能頁概念

| 項目 | 內容 |
|---|---|
| **目標** | 定義比價、推薦、酒商等關鍵功能頁的概念 |
| **關鍵任務** | ① 比價/搜索結果頁概念 ② 單品詳情頁概念（酒款解說 + 多商家價格對比 + AI 推薦理由）③ 場景推薦頁概念（送禮、約會、聚餐）④ 酒商店鋪頁概念 ⑤ 酒商入駐頁概念 ⑥ AI 選酒助手互動概念（受控 Top 3 推薦）|
| **產出** | 功能頁概念文件（每頁含目標、結構、關鍵文案方向） |

### WS-7：打磨與交付

| 項目 | 內容 |
|---|---|
| **目標** | 整合所有產出，形成完整可執行的品牌與網站概念包 |
| **關鍵任務** | ① 全局一致性審查 ② 調性校準（逐頁檢查 premium ↔ practical 平衡）③ 用戶視角走查 ④ 中英文案一致性審查 ⑤ 最終文件整合 |
| **產出** | 完整品牌概念包（Brand & Website Concept Package） |

## 4. 建議執行順序

```
Phase 1 ── 地基層
│
├── WS-1 品牌策略定位          ← 一切的前提
│   （必須先確定定位，否則後續所有決策都會搖擺）
│
└── WS-2 用戶旅程與信息架構     ← 與 WS-1 可部分並行
    （定位確認後，立即轉化為用戶視角的結構）

Phase 2 ── 骨架層
│
├── WS-3 品牌視覺系統          ← 依賴 WS-1 的調性結論
│
└── WS-4 首頁結構與故事線       ← 依賴 WS-1 + WS-2
    （視覺與結構可同步推進）

Phase 3 ── 血肉層
│
├── WS-5 文案撰寫              ← 依賴 WS-1 + WS-4
│
└── WS-6 核心功能頁概念         ← 依賴 WS-2 + WS-4

Phase 4 ── 打磨層
│
└── WS-7 整合與交付             ← 依賴以上全部
```

**為什麼這個順序：**
- 定位是所有創意的「憲法」——沒有定位就寫文案，等於蒙眼射箭
- 用戶旅程決定信息架構——架構錯了，再好的文案也救不回來
- 視覺和結構可以並行，因為它們各自依賴定位但彼此獨立
- 文案必須在結構確定後才寫，否則會反覆修改

## 7. 規劃階段的風險

| 風險 | 後果 | 應對 |
|---|---|---|
| **過早追求 premium 感，忽略實用性** | 網站很漂亮但用戶不知道這是幹什麼的，跳出率極高 | 每個 section 都必須有明確的功能目標，不能只有氛圍 |
| **定位模糊，雙線用戶都沒抓住** | 入門小白覺得太高端，愛好者覺得太淺 | 首頁分區明確，每個區塊有清晰的目標用戶 |
| **品牌語言太抽象** | 聽起來很美但說不清楚平台做什麼 | 堅持「每句話都能回答一個用戶問題」的原則 |
| **國際極簡風格過冷** | 視覺高級但缺乏人情味，與「朋友感」矛盾 | 用內容溫度和社區元素平衡視覺的冷感 |
| **中英雙語調性不統一** | 兩種語言像兩個不同的品牌 | 先定英文調性（與品牌名一致），再確保中文翻譯保持同等質感 |
| **AI 推薦邊界不清** | AI 輸出超出平台數據，推薦了不存在的酒款或錯誤信息 | 在品牌和技術層面都要設定嚴格的輸出邊界 |
| **信任機制設計不清** | 用戶不相信推薦是中立的，懷疑平台收了酒商的錢 | 價格透明 + 用戶口碑雙管齊下，品牌層面強調「不藏價」 |
| **酒商角色不清晰** | 酒商不知道自己是「供應商」還是「合作夥伴」 | 明確定義：酒商是有獨立展示空間的合作夥伴，非匿名供應商 |

## 8. 建議協作流程

```
Step 1 ── 品牌定位對齊
         提出 2-3 個定位方向（含品牌人格、調性、核心語句）
         選擇 / 修正 / 混合
         產出：確認版定位文件

Step 2 ── 用戶旅程與首頁結構
         基於定位，設計雙線用戶旅程與首頁 section map
         確認信息優先級、雙路徑（以酒找商/以商找酒）
         產出：確認版首頁結構

Step 3 ── 調性示範與視覺方向
         撰寫 2-3 段調性示範文案（不同溫度）
         配合國際極簡視覺方向的文字描述
         確認最終調性刻度
         產出：調性標尺 + 視覺方向

Step 4 ── 首頁文案撰寫
         按確認的結構，逐 section 撰寫完整文案（中英雙語）
         含主標題、副標題、正文、CTA
         逐段反饋
         產出：完整首頁文案 v1

Step 5 ── 功能頁概念
         比價頁、單品頁、酒商店鋪頁、入駐頁的結構與文案方向
         AI 選酒助手的交互概念
         產出：功能頁概念文件

Step 6 ── 整合打磨
         全局一致性審查（中英雙語 + 調性統一）
         最終調性校準
         產出：完整品牌概念包
```

## 9. 品牌定位文件（已確認）

### 品牌名稱
**Your Wine Book**（暫定）

### 定位語
- **英文**：Drink smarter. Discover more.
- **中文**：在香港，輕鬆喝對每一支。

### 定位方向
**聰明生活** — 平衡工具實用性和文化質感。品牌像一個既懂酒又懂生活的編輯朋友——幫你選對、幫你省錢、順便讓你長知識。

### 品牌人格
- **主體**：可靠的編輯 — 專業判斷力、有品味的篩選
- **輔助**：朋友感 — 輕鬆、有溫度、不端架子
- **文化角色**：葡萄酒世界的入口橋樑 — 風土人情、產區故事

### 調性刻度
**4/5 偏親切**
- 從容、有品味但不端架子
- 實用又有質感
- 像有教養的朋友在認真跟你聊
- 書面中文為主，保持國際感和一致性，不用廣東話口語

### 調性示範
> 「週五晚上約了朋友打邊爐，
> 帶支什麼酒好？
> 我們替你比過價、選好風格、連故事都準備好了。
> 你只需要帶上好心情。」

### 品牌禁忌清單
| 禁忌類型 | 具體表現 | 原因 |
|---|---|---|
| **過度專業術語** | 不用「單寧」「丹寧」「風土」作為主標題 | 可在詳細解說中出現，但必須配白話翻譯 |
| **推銷壓力感** | 不用「限時搶購」「今日必搶」「錯過就沒」 | 與「從容」的品牌調性矛盾 |
| **奢侈排他感** | 不用「尊享」「臻選」「僅為懂酒之人」 | 排斥入門用戶，與「輕鬆」和「橋樑」定位矛盾 |
| **廣東話口語** | 不用「啲」「嘅」「唔」等口語字 | 保持書面中文的國際感和跨地域一致性 |

### 視覺方向（文字描述）
- **風格**：國際極簡，乾淨、留白多、無襯線字體為主
- **光線**：明亮自然光，溫暖但不過度
- **場景**：生活場景（聚餐、送禮、獨飲、探索），非棚拍酒瓶
- **排版**：乾淨版式，雜誌感但不繁複
- **參考靈感**：Aesop / Seedlip 的視覺語言（僅視覺，非品牌模仿）
- **注意**：極簡中保留溫度，避免過冷

## 10. 用戶旅程與首頁結構（已確認）

### 核心使用場景（MVP 全部覆蓋）

### 導航結構
- **風格**：簡潔型，最多 4-5 個入口
- **組成**：Logo | 搜索框 | 探索酒款 | 酒商 | 關於我們
- **搜索框**：常駐導航欄，接入 AI 智能匹配 API

### 首頁主 CTA 策略
- **主 CTA 1**：搜索框 — 「搜索酒名、產區或風格...」（面向有目標的用戶）
- **主 CTA 2**：「不知道喝什麼？讓我們幫你選」（面向無目標的用戶）
- **兩者並列**，分流兩類用戶到不同路徑

### 單品詳情頁結構（分步露出）

**默認展示（核心信息）：**
- 酒名（中英文）+ 酒標圖片
- 風格標籤（如「果香豐富」「適合配紅肉」「新手友好」）
- 簡明風格描述（1-2 句白話）
- 多商家價格對比表（含商家名、價格、跳轉按鈕）
- 推薦理由（AI 或編輯生成）

**展開查看更多：**
- 產區故事 / 酒莊介紹
- 詳細品飲筆記（配白話翻譯）
- 用戶評價
- 適合場景推薦
- 相似酒款推薦

### 交易模式：平台內代管店鋪

**核心思路：** 不做跳轉，為每家酒商在平台內建代管店鋪。用戶感覺在酒商的店裡買，但體驗由平台統一控制。

**用戶體驗：**
1. 用戶在單品頁選擇某酒商的價格 → 點擊購買
2. 進入該酒商的平台內店鋪頁面完成下單
3. 頁面風格可配置（酒商可選用自己的 logo/配色，或用平台默認樣式）
4. 整個流程不離開 Your Wine Book

**訂單傳遞（先簡後繁）：**
- **MVP 階段**：平台捕獲訂單後，通過郵件 / WhatsApp 自動通知酒商
- **後期演進**：建設酒商後台，酒商登錄查看和管理訂單

**優勢：**
- 用戶體驗統一可控，成交率更高
- 平台掌握完整交易數據
- 為後續平台內支付、履約打下基礎

### 用戶旅程地圖

**路徑 A — 有目標的用戶（搜索路徑）：**
```
首頁搜索框 → AI 智能匹配結果 → 單品頁（價格對比）→ 選擇酒商 → 代管店鋪下單
```

**路徑 B — 無目標的用戶（發現路徑）：**
```
首頁「讓我們幫你選」→ AI 選酒助手（場景/預算/口味問答）→ Top 3 推薦 → 單品頁 → 下單
```

**路徑 C — 場景驅動的用戶：**
```
首頁場景卡片（如「聚餐配酒」）→ 場景篩選頁 → 推薦結果 → 單品頁 → 下單
```

**路徑 D — 逛一逛的用戶：**
```
首頁編輯精選 → 單品頁 → 下單 / 繼續瀏覽
```

### 酒商店鋪調性策略
- **統一模板**：所有酒商店鋪使用平台統一模板，保證體驗一致
- **可配置元素**：酒商 logo、主色調可自定義

- **語氣規範**：店鋪頁文案遵循平台整體調性，不由酒商自由撰寫

### 視覺方向（參考 Seedlip 調性）

**整體風格：** 參考 Seedlip 的視覺語言——自然、精緻、大量留白、溫暖的中性色調。但不複製其植物烈酒的品牌氣質，而是轉化為「葡萄酒生活方式」的視覺表達。

**色彩系統：**
| 角色 | 方向 | 說明 |
|---|---|---|
| 背景色 | 暖白 / 奶油色 / 淺米色 | 大面積使用，營造溫暖留白感 |
| 主色 | 深酒紅或墨綠 | 體現葡萄酒屬性，用於標題、按鈕、強調 |
| 點綴色 | 銅金色 / 暖金色 | 用於圖標、分隔線、hover 效果，增加精緻感 |
| 文字色 | 深灰 / 接近黑 | 正文用深灰而非純黑，降低視覺壓力 |
| 輔助色 | 淺灰 / 米灰 | 用於卡片背景、邊框、次要信息 |

**字體方向：**
- 英文：無襯線字體為主（如 Inter、DM Sans 類型），標題可考慮帶一點人文氣息的字體
- 中文：思源黑體 / Noto Sans CJK TC 或類似的現代中文無襯線字體
- 字重層次分明：大標題粗、正文常規、輔助信息細

**攝影風格：**
- 自然光為主，溫暖但不過曝
- 生活場景（餐桌、聚會、陽台、廚房），非棚拍酒瓶特寫
- 有人的氛圍但不一定露臉，重點在場景和情緒
- 配色與網站色調統一，避免過於鮮豔的圖片

**排版原則：**
- 大量留白，不擁擠
- 卡片式佈局，模塊化
- 信息層次清晰（大標題 → 副標題 → 正文 → 輔助信息）
- 按鈕和 CTA 用主色，乾淨有力

│                                          │
│  ──────────────────────                  │
│                                          │

**四個場景的專屬篩選器：**

| 聚餐配酒 | 聚餐類型 · 人數 · 預算 |

**AI 行為規範：**
